# Webbender - Quick Start Guide

## What's Been Set Up ✅

### a) Raw JavaScript Source of Truth
- **`src/webbender.js`** - Clean, readable, well-formatted source code (11KB)
- All code formatted consistently with Prettier

### b) CI/CD Pipeline
- **`.github/workflows/build.yml`** - Automated GitHub Actions workflow that:
  - Runs on every push to `main` branch
  - Checks code formatting with Prettier
  - Builds and minifies the bookmarklet
  - Creates releases when you push git tags
  - Auto-deploys to GitHub Pages
  - Notifies CDN for cache invalidation

### c) CSP-Unblocked Install + Updates
- **Self-contained bookmarklet** generated in `dist/bookmarklet.js`
  - No external `<script>` injection at runtime
  - Works on sites with strict Content Security Policy headers
- **Version checks** use hosted `version.json` for update notifications

### d) Refined UI
- Modern dark theme with better visual hierarchy
- Smooth hover effects and transitions
- Better button styling and spacing
- Improved label structure with proper accessibility
- Settings persistence using localStorage
- 2-column grid for themes and dialogs

### e) Code Formatting
- **Prettier** configuration (`.prettierrc`)
- npm scripts:
  - `npm run format` - Auto-fix code formatting
  - `npm run format:check` - Check if formatting is needed
  - Runs automatically before build in CI/CD

## Quick Commands

```bash
# Install dependencies
npm install

# Build the bookmarklet
npm run build

# Format code
npm run format

# Check formatting without fixing
npm run format:check

# Watch for changes and rebuild
npm run watch
```

## Installation for Users

### CSP-Unblocked Version (Recommended)
Users should install from [https://webbender.web.app](https://webbender.web.app) by dragging the bookmarklet button or copying the generated code.

The installed bookmarklet is self-contained and does not inject external scripts, so CSP does not block execution.

## Making a Release

1. Update version in `package.json`
2. Run: `npm run format && npm run build`
3. Update `CHANGELOG.md`
4. Commit: `git add . && git commit -m "v1.x.x"`
5. Tag: `git tag v1.x.x && git push origin v1.x.x`
6. GitHub Actions automatically builds and releases!

- Users should save important edits with Ctrl+S before reloading
- Code is automatically formatted and tested before each release
- Update checks happen silently in background (once per 24h)
- Users see friendly notification when updates are available
